codeindex

Zero-install, deterministic repo-indexing engine — one vendorable file, a library, a CLI, and 26 MCP tools.

Walks a repo, extracts symbols and imports across 15 languages, resolves cross-file references into a typed link-graph, and renders byte-identical artifacts every time it runs. Vendor the single engine.mjs, install it from npm or Homebrew, or run it from a container — nothing else to install, ever.

Benchmarks

Measured against 01x-in/codeindex, universal-ctags and scip-typescript with a reproducible harness (scripts/bench/) — median of 5 runs, one warmup discarded. Every number on this page comes from the same site/benchmarks.json committed to this repo; nothing here is rounded or typed in by hand. Full methodology and every scenario: BENCHMARKS.md.

  • ctags is faster, on purpose universal-ctags wins cold-index time in every repo below — it only extracts flat definitions from a regex/parser pass. No call graph, no import resolution, no cross-file links; a tags file, not an index.
  • 01x's warm reindex wins, honestly 01x-in/codeindex's live SQLite model reindexes one touched file in 11 ms on vercel/next.js, against codeindex's 1,828 ms for the same touch. codeindex renders static, byte-stable artifacts on every run instead of mutating a database — a real architectural tradeoff, not a bug to fix.
  • the next.js cold-init number sits at a timeout edge 01x's cold init on vercel/next.js runs 85–125 s on this machine — close to this harness's 120 s per-process spawn timeout. The 86,062 ms figure below is an authentic measurement from this session, not a clean average; read it next to that boundary, and next to 01x-in's own published 121,037 ms for the same operation on their machine, their SHA.
  • n/a means measured-and-failed, never zero scip-typescript and 01x-in both failed to cold-index 2 of 7 repos this session (npm install failed / no tsconfig) — shown as n/a (reason), exactly as the harness recorded it.

Cold index — vercel/next.js (20,000 files)

Full process spawn per run into a fresh output dir. Files capped at codeindex's own default walk limit (20,000) — the repo has more source files on disk than that cap; hover or focus a bar for the exact reading.

Fig. 02 — capabilities

Features

Everything below ships in the same zero-dependency bundle — no feature is an optional add-on package.

Keyless search + deterministic fuzzy

BM25 lexical ranking over symbol names, path segments and headings. A term with zero document frequency falls back to a byte-deterministic character-trigram match (Dice ≥ 0.6) — typo tolerance with no embeddings. Disable with --no-fuzzy.

Deterministic semantic tier

An opt-in static token → vector lookup table — no neural forward pass. Encode → mean-pool → L2-normalize → round-half-to-even int8 quantize; ranking is a pure integer dot product, byte-identical everywhere. No model on disk, no behavior change.

Rich embedding tier — v2.11.0

Point at a containerized HTTP endpoint (ghcr.io/maxgfr/codeindex-embed, all-MiniLM-L6-v2) via CODEINDEX_EMBED_ENDPOINT. Wins over the static tier by explicit intent; degrades to lexical on exit 0 if unreachable — never a silent fallback to the wrong model.

Call graph & caller index

Typed import / call / use / doc-link / mention edges at file and module level. Per-symbol callers and references resolve in-proc in sub-millisecond time on an already-loaded warm scan.

Graph analytics

Louvain communities, PageRank & betweenness centrality, dead-code detection, complexity × churn hotspots, change coupling, surprise-edge detection, and a tests→code map — all derived from the same link-graph.

SCIP export

index.scip via a hand-rolled zero-dependency protobuf encoder — validated against the official scip CLI's stats/lint.

Multi-ecosystem workspaces

npm/yarn, pnpm, lerna, nx, cargo, go.work, maven, gradle, uv and Composer — detected and named without running any of their own tooling.

Symbolic edits over MCP

replace_symbol_body, insert_after_symbol, insert_before_symbol — precise edits addressed by symbol name, not by line ranges that drift.

mermaid & repo map

Render a mermaid dependency diagram or a plain-text repo map directly from the graph — no separate diagramming step.

Byte-deterministic outputs

Two cold builds of an unchanged repo produce identical graph.json/symbols.json — confirmed on 7 of 7 measured repos this session. 01x's SQLite DB embeds ids and timestamps, so it isn't byte-comparable by design.

Fig. 03 — usage

Quickstart

Five ways in — pick the one matching your integration.

Install it as a regular dependency and import the pure functions.

npm i @maxgfr/codeindex
import { scanRepo, ENGINE_VERSION } from "@maxgfr/codeindex";

const scan = scanRepo("/path/to/repo");

Skills and CLI tools that don't want an npm dependency should still prefer vendoring — see the Vendoring tab.

The CLI ships in the same package as the library.

# or: brew install maxgfr/tap/codeindex
npm i -g @maxgfr/codeindex

codeindex index   --repo . --out .codeindex
codeindex graph   --repo . > graph.json
codeindex scip    --repo . --out index.scip
codeindex callers --repo .
codeindex search  "http client retry" --repo .

Serves the engine over stdio: 26 tools, including scan_summary, graph, symbols, find_symbol, find_references, callers, search, repo_map, hotspots, coupling, dead_code, complexity, mermaid, grep, replace_symbol_body, insert_after_symbol, insert_before_symbol, memory tools, workspaces, churn, check_rules and embed_status. 

claude mcp add codeindex -- codeindex mcp
{
  "mcpServers": {
    "codeindex": {
      "command": "codeindex",
      "args": ["mcp"]
    }
  }
}

Multi-arch (linux/amd64, linux/arm64) — nothing inside but node and the bundle, no npm install. 

docker run --rm -v "$PWD":/work ghcr.io/maxgfr/codeindex scan --repo /work
docker run --rm -v "$PWD":/work ghcr.io/maxgfr/codeindex index --repo /work --out /work/.codeindex
docker run -i --rm -v "$PWD":/work ghcr.io/maxgfr/codeindex mcp

Pin by digest in CI or anywhere reproducibility matters: ghcr.io/maxgfr/codeindex@sha256:…. The rich semantic-search tier ships separately as ghcr.io/maxgfr/codeindex-embed.

Consumers who don't want an npm dependency vendor scripts/engine.mjs + engine.d.mts at a pinned release tag instead — the model ultraindex, ultradoc, ultrasec, ultraeval, reconstruct, construct and ultra11y all use, so each stays a standalone, single-file install. 

node scripts/sync-engine.mjs --ref v2.11.0

Full steps and version-constant semantics: docs/MIGRATION.md.
